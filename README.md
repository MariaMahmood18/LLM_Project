## Track
Track A: Research-Oriented Project

## Project Overview
This project evaluates the robustness of LLM-based radiology report 
summarization under simulated OCR-style text noise. We investigate 
whether retrieval-augmented generation (RAG) can improve stability 
and factual consistency when input text contains character-level 
perturbations typical of OCR digitization errors.

## Dataset
Indiana University Chest X-Ray Reports (OpenI)
- 4000 radiology reports with XML structure
- Contains findings and impression sections
- Publicly available under CC BY-NC-ND 4.0 license
- URL: https://openi.nlm.nih.gov/

## Reproduce Results

### Run the full evaluation (requires GPU, real BART)
```bash
python scripts/run_evaluation.py
```
This runs: noise sweep (5/10/15/20%), RAG top-k ablation (k=1,3,5), 
failure taxonomy classification, and saves 5 plots to `artifacts/plots/`.

### Run the lightweight A3 diagnostics (no GPU needed)
```bash
python scripts/run_a3_diagnostics.py
```
Uses a rule-based mock summarizer instead of BART. Good for testing 
the pipeline structure locally. Results saved to `artifacts/logs/`.

| Condition | Noise | ROUGE-L | BERTScore |

### Output files explained

| File | What it contains |
|------|-----------------|
| `artifacts/logs/a2_results.json` | A2 aggregate scores (3 conditions) |
| `artifacts/logs/evaluation_results.json` | A3 full sweep (12 conditions) |
| `artifacts/logs/failure_taxonomy.json` | Failure category counts |
| `artifacts/logs/noisy_baseline_noise10.json` | Per-sample predictions at 10% noise |
| `artifacts/logs/rag_top1_noise10.json` | Per-sample predictions, RAG k=1 |
| `artifacts/plots/` | 5 evaluation plots (generated by run_evaluation.py) |

## Configuration

Edit `configs/default.yaml` to change:
- `data.max_samples`: Number of reports to use (reduce for speed)
- `noise.level`: OCR noise level (0.05, 0.10, 0.15, 0.20)
- `model.name`: HuggingFace model to use
- `rag.top_k`: Number of retrieved examples for RAG
